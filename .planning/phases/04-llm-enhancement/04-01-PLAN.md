  <action>
Add `import openai` at the top of enrich_calendar.py (install openai package: `pip install openai`).

Create function `generate_llm_content(idea, examples, audio, client=None)` that:

1. **Client setup:** If client is None, create `openai.OpenAI()` which reads OPENAI_API_KEY from env automatically. Accept client param for testability.

2. **Build a single prompt** that generates all fields in one API call. The prompt must include:
   - System message: "You are a hype coach content strategist for Logara, a startup documenting its founder journey. You give energetic, direct, filmable content direction. Respond in valid JSON only."
   - User message containing:
     - Content Type and Topic from the idea
     - Audio info: audio_title, audio_author, audio_confidence
     - For each example (up to 3): caption, views, likes, author_username, audio_title
   - Requested JSON output format:
     ```
     {
       "audio_fit_reason": "one sentence about the audio's vibe and usage potential",
       "hook_summaries": ["2 sentences for ex1", "2 sentences for ex2", "2 sentences for ex3"],
       "remix_ideas": ["bullet 1", "bullet 2", "bullet 3"]
     }
     ```

3. **Prompt rules from CONTEXT.md:**
   - audio_fit_reason: General vibe description of the audio, NOT tied to the specific content idea. Example: "Trending motivational track with 1.5M uses, high energy, good for visual transitions"
   - hook_summaries: Context-aware, referencing what the example video actually did (from caption). Two sentences each: what they did + why it works/what to learn.
   - remix_ideas: 2-3 one-liner bullets, filmable and concrete (actions not abstract advice), tailored to the content Type (Teach = educational angles, Story = narrative arcs, BTS = behind-the-scenes framing, etc.)
   - Brand context: Logara is a startup; founder documenting the building journey; hopecore-light tone (optimistic, authentic, not forced)
   - Audio confidence tone adjustment: If audio_confidence is "high", use assertive language in audio_fit_reason ("Use this track - ..."). If "low", use suggestive language ("Consider this audio, but feel free to pick your own - ...")

4. **API call:** Use `client.chat.completions.create()` with:
   - model: "gpt-4o-mini"
   - response_format: {"type": "json_object"}
   - temperature: 0.8 (creative but not wild)
   - max_tokens: 500

5. **Wrap the API call with _call_with_retry()** for resilience. Create a small inner function that makes the actual API call, pass it to _call_with_retry. Note: _call_with_retry catches requests.RequestException and ValueError. OpenAI errors inherit from openai.APIError. Update _call_with_retry's except clause to ALSO catch `openai.APIError` so retries work for OpenAI failures too.

6. **Parse response:** json.loads the response content. Extract fields with .get() defaults:
   - audio_fit_reason: str, default ""
   - hook_summaries: list of str, default []
   - remix_ideas: list of str, default []

7. **Format remix_ideas for Excel:** Join bullets with newline, prefixing each with a bullet character. Example: "- Film yourself opening laptop at 5am\n- Cut to the product demo with voiceover\n- End with the metrics reveal"

8. **Return dict:**
   ```python
   {
     "audio_fit_reason": str,
     "ex1_hook_summary": str,  # hook_summaries[0] or ""
     "ex2_hook_summary": str,  # hook_summaries[1] or ""
     "ex3_hook_summary": str,  # hook_summaries[2] or ""
     "remix_ideas": str,       # formatted bullet string
   }
   ```
   If the API call fails completely (_call_with_retry returns []), return all fields as empty strings.

Also add `"openai"` to the requirements. Run `pip install openai` to verify it installs.
