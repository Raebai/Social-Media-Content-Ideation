  <action>
Create `requirements.txt` with initial dependencies:
```
openpyxl>=3.1.0

```

Create `enrich_calendar.py` with the following structure:

1. **ContentIdea dataclass** with fields:
   - `row_number: int` (original Excel row number, 1-indexed including header)
   - `date: datetime.date` (parsed date)
   - `content_type: str` (validated, title-cased Type)
   - `topic: str` (cleaned, sentence-cased)
   - `description: str` (cleaned, sentence-cased)
   - `idea_text: str` (constructed from type, topic, description)

2. **normalize_text(text: str) -> str** function:
   - Replace U+2011 (non-breaking hyphen) with regular hyphen
   - Replace U+00A0 (non-breaking space) with regular space
   - Strip leading/trailing whitespace
   - Collapse multiple internal spaces into single space
   - Return cleaned string

3. **sentence_case(text: str) -> str** function:
   - Capitalize first character, lowercase the rest? No -- sentence case means only capitalize the first letter, leave the rest as-is (preserve proper nouns, acronyms). Implementation: `text[0].upper() + text[1:]` if text is non-empty.

4. **parse_date(date_str: str) -> datetime.date** function:
   - First normalize the text (handles U+2011 hyphens)
   - Parse using dateutil.parser.parse().date()
   - Raise ValueError with helpful message if parsing fails

5. **VALID_TYPES** constant: `{"Story", "BTS", "Teach", "Trend", "Breakdown", "Reflection", "Depth"}`

6. **load_content_ideas(file_path: str) -> list[ContentIdea]** function:
   - Open workbook with openpyxl (read_only=True, data_only=True)
   - Try to get sheet named "Sheet1"; if not found, fall back to first sheet (log which sheet used)
   - Read header row (row 1). Validate all 5 required columns exist: Day, Date, Type, Topic, Description
     - Use case-insensitive matching for column names
     - If any column missing: raise ValueError with detailed message showing expected columns vs found columns
   - Map column positions from header
   - Iterate data rows (row 2+):
     - Extract values for Date, Type, Topic, Description (skip Day -- not carried forward)
     - Normalize all text values with normalize_text()
     - Check if Type, Topic, or Description is empty/None -- if so, log "Skipping row {N}: missing {field}" and continue
     - Validate Type (title-cased) against VALID_TYPES -- if invalid, raise ValueError with message showing the bad Type value, row number, and valid options
     - Sentence-case Topic and Description
     - Title-case Type
     - Parse date with parse_date()
     - Construct idea_text: `f"{content_type} | {topic} | {description}"`
     - Create ContentIdea and append to list
   - Close workbook
   - Return list of ContentIdea objects

7. **print_summary(ideas: list[ContentIdea], skipped: list[dict])** function:
   - Print "Loaded {N} content ideas"
   - Print "Types: {comma-separated unique types}"
   - Print "Date range: {earliest} to {latest}"
   - If skipped rows exist, print each: "Skipped row {row_number}: missing {field}"

8. **if __name__ == "__main__"** block:
   - Call load_content_ideas("Content ideas.xlsx")
   - Call print_summary()
   - This serves as a quick smoke test during development

IMPORTANT details:
- Do NOT use pandas -- use openpyxl directly (lighter dependency for reading)
- The skipped rows tracking should be done inside load_content_ideas and returned alongside ideas (return a tuple, or track via a second return value)
- All print output should go to stdout
- Error messages must be specific: show row numbers, field names, actual vs expected values
