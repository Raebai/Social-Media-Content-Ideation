**1. normalize_result(raw: Dict[str, Any]) -> Optional[Dict[str, Any]]**

Maps Apify output schema to our standard schema. Field mapping:
- `video_id` = raw["id"] (string)
- `url` = raw["webVideoUrl"] (string)
- `caption` = raw.get("text", "") (string)
- `author_username` = raw.get("authorMeta", {}).get("name", "") (string)
- `create_time` = raw.get("createTimeISO", None) (string ISO or None)
- `views` = raw.get("playCount", 0) (int, default 0)
- `likes` = raw.get("diggCount", 0) (int, default 0)
- `comments` = raw.get("commentCount", 0) (int, default 0)
- `shares` = raw.get("shareCount", 0) (int, default 0)
- `audio` = sub-dict:
  - `audio_id` = raw.get("musicMeta", {}).get("musicId", "")
  - `title` = raw.get("musicMeta", {}).get("musicName", "")
  - `author` = raw.get("musicMeta", {}).get("musicAuthor", "")
  - `url` = raw.get("musicMeta", {}).get("playUrl", "")

Return None if raw is missing "id" key or "id" is falsy. This gracefully skips malformed results.

**2. deduplicate_results(results: List[Dict[str, Any]]) -> List[Dict[str, Any]]**

Iterate through results, track seen video_ids in a set, keep only first occurrence of each video_id. Return the deduplicated list preserving original order.

**3. filter_old_results(results: List[Dict[str, Any]], max_age_days: int = 120) -> List[Dict[str, Any]]**

Filter out videos older than max_age_days. For each result:
- If create_time is None or empty string, KEEP the video (do not reject on missing data)
- Otherwise, parse create_time with dateutil.parser.parse, compare to datetime.datetime.now(datetime.timezone.utc)
- If age > max_age_days, exclude
- If parsing fails, KEEP the video (graceful fallback)

Return filtered list.

**1. score_video(video: Dict[str, Any], keywords: List[str] = None) -> Dict[str, Any]**

Computes engagement score and returns the video dict with added score fields.

Step 1 - Base score (DAT-04):
```python
base = (math.log10(video.get("views", 0) + 1) * 0.65 +
        math.log10(video.get("likes", 0) + 1) * 0.25 +
        math.log10(video.get("comments", 0) + 1) * 0.10)
```

Step 2 - Recency boost (DAT-05):
- Parse create_time if present. Calculate days_old from now.
- If days_old <= 14: recency_boost = 0.2
- If days_old <= 30: recency_boost = 0.1
- Otherwise (or if create_time missing/unparseable): recency_boost = 0.0

Step 3 - Relevance boost (DAT-06):
- If keywords provided and caption exists:
  - caption_lower = caption.lower()
  - count overlapping keywords: sum(1 for kw in keywords if kw.lower() in caption_lower)
  - relevance_boost = min(overlap_count * 0.05, 0.2)
- Otherwise: relevance_boost = 0.0

Step 4 - Final score (DAT-07):
```python
final_score = base + recency_boost + relevance_boost
```

Add to the video dict:
- `base_score` = base
- `recency_boost` = recency_boost
- `relevance_boost` = relevance_boost
- `final_score` = final_score

Return the modified video dict.

**2. process_results(raw_results: List[Dict[str, Any]], keywords: List[str] = None) -> List[Dict[str, Any]]**

Orchestrates the full pipeline:
1. Normalize: `[normalize_result(r) for r in raw_results]`, filter out None
2. Deduplicate: `deduplicate_results(normalized)`
3. Filter old: `filter_old_results(deduped)`
4. Score: `[score_video(v, keywords) for v in filtered]`
5. Sort: by `final_score` descending
6. Return sorted list

The keywords parameter is optional. When provided, it enables relevance boosting. Callers will extract keywords from the ContentIdea topic/description (same extract_keywords logic from generate_queries).
